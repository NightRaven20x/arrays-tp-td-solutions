Propositional Logic & Connectives Foundations

This tab covers propositional logic, truth tables, equivalences, and basic applications in reasoning and computer science. [file:3][file:4]

What is this?

What is a Proposition?

A proposition is a declarative statement that is either true or false, but not both. [file:3]

Propositional logic combines simple propositions using logical connectives to build more complex statements whose truth values are determined systematically. [file:3][file:4]

Important: questions, commands, or expressions with unspecified variables (like “x + 5 = 3”) are not propositions because they do not have a fixed truth value. [file:3]
When to use it?

Applications of Propositional Logic

  • Describing and reasoning about conditions in algorithms, programs, and specifications. [file:3]
  • Designing and verifying digital circuits using Boolean expressions and truth tables. [file:4]
  • Formalising rules in AI systems, theorem provers, and expert systems like MYCIN. [file:2][file:4]
Example: “If you are older than 13 or with your parents, then you can attend a PG‑13 movie” can be written as \((A \lor B) \to C\). [file:4]
Core concepts

Logical Connectives at a Glance

Connective Symbol Meaning (informal) True when…
Negation \(\lnot p\) “not p” p is false. [file:3]
Conjunction \(p \land q\) “p and q” both p and q are true. [file:3]
Disjunction (inclusive) \(p \lor q\) “p or q or both” at least one of p, q is true. [file:3]
Exclusive or \(p \oplus q\) “either p or q, but not both” exactly one of p, q is true. [file:3]
Implication \(p \to q\) “if p then q” false only when p is true and q is false. [file:3]
Biconditional \(p \leftrightarrow q\) “p if and only if q” p and q have the same truth value. [file:3]
Key truth-table fact: an implication \(p \to q\) is logically equivalent to \(\lnot p \lor q\). [file:5]
Step-by-step guide

How to Build a Truth Table for a Compound Formula

Problem type

Constructing the truth table of a compound proposition such as \((p \to q) \land (\lnot p \leftrightarrow q)\). [file:4]

Step-by-step:

  1. Identify all distinct elementary propositions (here: p, q). [file:4]
  2. Determine the number of rows: use \(2^n\) where n is the number of propositions (here \(2^2 = 4\) rows). [file:4]
  3. Create columns for each elementary proposition and for all intermediate sub‑formulas (e.g., \(\lnot p\), \(p \to q\), \(\lnot p \leftrightarrow q\)). [file:4]
  4. Fill in the columns for p and q with all combinations of T and F. [file:4]
  5. Compute the truth value of each intermediate column using the definitions in the table above. [file:3][file:4]
  6. Finally compute the value of the target formula using the intermediate columns for each row. [file:4]
  7. Analyse the “target” column: if all entries are T, the formula is a tautology; if all F, a contradiction; otherwise a contingency. [file:4]
Worked Example: Build the table for \((p \land q) \to p\). List columns p, q, \(p \land q\), \((p \land q) \to p\) and evaluate row by row; the final column is always T, so the formula is a tautology. [file:5]
Common mistake: forgetting that inclusive “or” (\(\lor\)) is true when both operands are true, unlike exclusive “or” (\(\oplus\)). [file:3]
Theorems & rules

Important Logical Equivalences

De Morgan’s Laws: \(\lnot(p \lor q) \leftrightarrow (\lnot p \land \lnot q)\) and \(\lnot(p \land q) \leftrightarrow (\lnot p \lor \lnot q)\). [file:5]
  • Double negation: \(\lnot(\lnot p) \leftrightarrow p\). [file:5]
  • Distributive: \(p \lor (q \land r) \leftrightarrow (p \lor q) \land (p \lor r)\). [file:5]
  • Implication: \(p \to q \leftrightarrow (\lnot p \lor q)\). [file:5]
Use these equivalences to simplify formulas and to transform them into forms better suited for proofs. [file:5]
Common pitfalls

What to Avoid

  • Treating \(p \to q\) as symmetric; the converse \(q \to p\) is not logically equivalent in general. [file:3]
  • Confusing a statement with its inverse \(\lnot p \to \lnot q\); only the contrapositive \(\lnot q \to \lnot p\) is equivalent. [file:3]
  • Mixing up inclusive and exclusive “or” in everyday language translations. [file:3][file:4]

Predicate Logic & Quantifiers Structure

This tab upgrades propositional logic with variables, predicates and quantifiers to reason about groups of objects. [file:5][file:7]

What is this?

Predicates and Terms

Predicate logic introduces objects (constants and variables) and predicates that express properties or relations among these objects. [file:5][file:7]

A predicate like \(P(x)\) becomes a proposition only after substituting a specific object for x, such as \(P(2)\) meaning “2 is prime”. [file:5][file:7]

Example: Let Student(x) mean “x is a student”. Student(John) can be true, Student(Jane) can be false, depending on each person. [file:5][file:7]
When to use it?

Why Predicate Logic?

  • To express general rules like “all CS graduates passed CS441” without listing every graduate. [file:5][file:7]
  • To talk about existence, such as “some student graduates with honors”. [file:5][file:7]
  • To specify properties of data structures or systems in computer science (e.g., invariants). [file:5]
Predicate logic solves the limitation of propositional logic where repeated statements for many objects had to be enumerated one by one. [file:5][file:7]
Quantifiers

Universal vs Existential Quantifier

Quantifier Notation Meaning True when…
Universal \(\forall x P(x)\) “for all x, P(x) holds” P(x) is true for every element in the domain. [file:5][file:7]
Existential \(\exists x P(x)\) “there exists x such that P(x) holds” P(x) is true for at least one element in the domain. [file:5][file:7]
Truth characterisation (finite domain): \(\forall x P(x)\) is equivalent to \(P(x_1) \land P(x_2) \land \dots \land P(x_N)\), while \(\exists x P(x)\) is equivalent to \(P(x_1) \lor P(x_2) \lor \dots \lor P(x_N)\). [file:7]
Step-by-step guide

Translating English Sentences with Quantifiers

Problem type

Turn an English statement about many objects into a precise predicate‑logic formula. [file:5][file:7]

Step-by-step:

  1. Choose the universe of discourse (e.g., “all people”, “all real numbers”). [file:5][file:7]
  2. Define predicates for properties and relations you need (e.g., Student(x), at(x,Upitt), Smart(x)). [file:5][file:7]
  3. Identify if the sentence speaks about “all”, “none” (universal) or “some”, “at least one” (existential). [file:5][file:7]
  4. Introduce variables and choose the right quantifier (∀ or ∃) with those variables. [file:5][file:7]
  5. Write the inner logical structure using →, ∧, ∨, ¬ to match the English meaning. [file:5][file:7]
  6. Simplify if possible (e.g., typical pattern: “All S are P” becomes \(\forall x (S(x) \to P(x))\)). [file:7]
Example: “All Upitt students are smart”. If the domain is all people and Student(x) means “x is a student”, at(x,Upitt) means “x is at Upitt”, Smart(x) means “x is smart”, the translation is \(\forall x (Student(x) \land at(x,Upitt) \to Smart(x))\). [file:7]
Example (existential): “Some Upitt student is smart” becomes \(\exists x (Student(x) \land at(x,Upitt) \land Smart(x))\). [file:7]
Common mistake: using ∧ instead of → for universal statements; “All S are P” should be \(\forall x (S(x) \to P(x))\), not \(\forall x (S(x) \land P(x))\). [file:7]
Nested quantifiers

Order of Quantifiers

When different kinds of quantifiers are nested, the order matters: \(\forall x \exists y\,L(x,y)\) is not equivalent to \(\exists y \forall x\,L(x,y)\). [file:7]

Example: \(\forall x \exists y\,L(x,y)\) can mean “every person loves someone”, while \(\exists y \forall x\,L(x,y)\) means “there is someone whom everybody loves”. [file:7]
If all quantifiers are of the same type (all ∀ or all ∃), the order does not change the meaning. [file:7]
Negation

Negating Quantified Statements

De Morgan laws for quantifiers: \(\lnot \exists x P(x) \leftrightarrow \forall x \lnot P(x)\) and \(\lnot \forall x P(x) \leftrightarrow \exists x \lnot P(x)\). [file:6]
Example: “Nothing is perfect” can be written as \(\lnot \exists x\,Perfect(x)\) or equivalently as \(\forall x\,\lnot Perfect(x)\). [file:6]
When pushing negation inside, always flip ∀ to ∃ and ∃ to ∀, and negate the predicate formula. [file:6]

Sets, Operations & Cartesian Products Algebra I

This tab focuses on basic set operations, Cartesian products, and simple algebraic exercises involving subsets of \(\mathbb{R}\). [file:2]

What is this?

Sets and Basic Operations

A set is a collection of distinct elements, often subsets of \(\mathbb{R}\) such as finite sets, intervals, or regions in the plane. [file:2]

Standard operations on sets include union, intersection, complement, and Cartesian product. [file:2]

Example: \(A_1 = \{1,2,3\}\), \(A_2 = [1,3[\), \(A_3 = ]2,4]\), \(A_4 = \{0,1,2,3\}\) are subsets of \(\mathbb{R}\) combining discrete elements and intervals. [file:2]
When to use it?

Why Set Operations Matter

  • To describe domains and constraint regions in algebra and analysis. [file:2]
  • To build relations and functions via Cartesian products. [file:2]
  • To reason about combinations of conditions (e.g., solving inequalities that define subsets of \(\mathbb{R}\)). [file:2]
Step-by-step guide

How to Compute Unions, Intersections & Cartesian Products

Problem type

Given sets \(A_i\) in \(\mathbb{R}\), compute \(A_i \cup A_j\), \(A_i \cap A_j\), complements, and products \(A_i \times A_j\). [file:2]

Step-by-step for union/intersection:

  1. Write down explicit descriptions of each set (elements or interval bounds). [file:2]
  2. For unions, include elements or points that belong to at least one set; for intersections, keep only elements common to all sets involved. [file:2]
  3. For intervals, reason on the number line about overlaps, endpoints, and openness/closedness. [file:2]
  4. Simplify the resulting description, merging adjacent intervals when appropriate. [file:2]

Step-by-step for Cartesian product:

  1. Recall definition: \(A \times B = \{(a,b) \mid a \in A,\; b \in B\}\). [file:2]
  2. If sets are finite, list all ordered pairs explicitly. [file:2]
  3. If sets are intervals, describe the product as a region in \(\mathbb{R}^2\) (e.g., rectangle). [file:2]
  4. For intersections like \((A \times B) \cap (C \times D)\), use the identity \((A \times B) \cap (C \times D) = (A \cap C) \times (B \cap D)\) when valid. [file:2]
Example: To show \((A \times B) \cap (C \times D) = (A \cap C) \times (B \cap D)\), take a generic pair \((x,y)\) in the left side and prove it must satisfy \(x \in A \cap C\) and \(y \in B \cap D\), and conversely. [file:2]
Common mistake: assuming \(A \times B = B \times A\); in general, the order of coordinates changes the set of ordered pairs. [file:2]
Inequalities in sets

Sets Defined by Inequalities

Some exercises define sets \(E_i = \{x \in A_i \mid x^2 - 7x + 10 \le 0\}\), which require solving a quadratic inequality on each domain \(A_i\). [file:2]

Step-by-step solving quadratic inequalities on a set:

  1. Factor the polynomial if possible (e.g., \(x^2 - 7x + 10 = (x-2)(x-5)\)). [file:2]
  2. Find roots and sign intervals of the product. [file:2]
  3. Determine where the expression is ≤ 0 or ≥ 0 using a sign chart. [file:2]
  4. Intersect this solution set with the given domain \(A_i\) to obtain \(E_i\). [file:2]
Example: If the solution of \(x^2 - 7x + 10 \le 0\) is \([2,5]\), then for \(A_1 = \{1,2,3\}\) we get \(E_1 = \{2,3\}\) after intersecting. [file:2]

Functions, Images & Inverses Mappings

This tab covers basic function concepts: images, preimages, injectivity, surjectivity, bijections and compositions. [file:2]

What is this?

Functions Between Sets

A function \(f : A \to B\) assigns each element of A exactly one element of B, called its image. [file:2]

For a subset \(X \subseteq A\), the direct image \(f(X)\) and inverse image \(f^{-1}(Y)\) for \(Y \subseteq B\) are central in problem solving. [file:2]

When to use it?

Why Function Analysis?

  • To characterise solution sets via images and preimages. [file:2]
  • To study injectivity/surjectivity (one‑to‑one, onto) for solving equations and building inverses. [file:2]
  • To analyse compositions \(g \circ f\) in structured domains like \(\mathbb{R}\) and \(\mathbb{R}^2\). [file:2]
Step-by-step guide

Computing Direct and Inverse Images

Problem type

Given \(f : \mathbb{R} \to \mathbb{R}\) and a set \(A \subseteq \mathbb{R}\), find \(f(A)\) and \(f^{-1}(B)\) for some subset \(B\). [file:2]

Step-by-step for direct image \(f(A)\):

  1. Write the expression for \(f(x)\). For instance, \(f(x) = |x| - 1\). [file:2]
  2. Describe A (e.g., interval \([-1,4]\)). [file:2]
  3. Analyse \(f(x)\) on A: split into pieces if needed (e.g., absolute value requires separating negative and non‑negative parts). [file:2]
  4. Determine the range values attained on A and express \(f(A)\) as intervals or unions. [file:2]

Step-by-step for inverse image \(f^{-1}(B)\):

  1. Start from condition \(f(x) \in B\). Rewrite as inequalities or equations in x. [file:2]
  2. Solve these conditions to obtain allowed x values. [file:2]
  3. Intersect the solution with the domain of f if needed. [file:2]
  4. Express \(f^{-1}(B)\) as subsets (intervals or unions) of the domain. [file:2]
Example: For \(f(x) = |x| - 1\) and \(A = [-1,4]\), first compute f on \([-1,0]\) and on \([0,4]\) separately to obtain the complete image. [file:2]
Injective / Surjective

Checking Injectivity and Surjectivity

A function is injective if different inputs give different outputs, and surjective if every element in the codomain is attained as some output. [file:2]

Step-by-step to test injectivity over \(\mathbb{R}\):

  1. Solve \(f(x_1) = f(x_2)\) and check whether this forces \(x_1 = x_2\). [file:2]
  2. If not, restrict the domain (e.g., to an interval) where the function becomes strictly monotone. [file:2]
Example: For \(f(x) = \dfrac{x}{1 + x^2}\), the function is not injective on all \(\mathbb{R}\) but becomes injective when restricted to \(]1,+\infty[\) where it is strictly decreasing. [file:2]
Compositions

Understanding \(g \circ f\) and \(f \circ g\)

Given \(f : E \to F\) and \(g : F \to G\), the composition \(g \circ f\) maps x in E to \(g(f(x))\) in G, and order matters. [file:2]

Example: For \(E = \mathbb{R}, F = ]-1,1[, G = \mathbb{R}\), \(f(x) = \frac{x}{1+|x|}\), \(g(x) = \sqrt{1-x}\), \(g \circ f\) is defined only where \(1 - f(x) \ge 0\) and \(|f(x)| \le 1\). [file:2]
Always check that the image of the inner function lies inside the domain of the outer function before composing. [file:2]
Inverse functions

Finding the Inverse of a Bijective Function

A bijective function is both injective and surjective, which guarantees the existence of a unique inverse function. [file:2]

Step-by-step to compute \(f^{-1}\):

  1. Write \(y = f(x)\) and treat y as the output variable. [file:2]
  2. Solve this equation for x in terms of y, ensuring the solution respects the original domain/codomain restrictions. [file:2]
  3. Rename x ← \(f^{-1}(y)\), and clearly specify the domain of \(f^{-1}\) (equal to the codomain of f). [file:2]
  4. Verify \(f(f^{-1}(y)) = y\) and \(f^{-1}(f(x)) = x\) for all allowed x and y. [file:2]
Example: For \(f(x) = \frac{x}{1 + |x|}\) from \(\mathbb{R}\) to \(]-1,1[\), once bijectivity is established, solve \(y = \frac{x}{1 + |x|}\) for x in terms of y to derive \(f^{-1}\). [file:2]

Proof Methods & Theorems Reasoning

This tab organises formal and informal proof techniques: direct, indirect, contradiction, cases, and equivalence proofs. [file:1][file:6]

What is this?

Theorems and Proofs

A theorem is a statement that can be shown to be true by a logically valid argument using premises, axioms, and already proven results. [file:1][file:6]

Formal proofs derive the conclusion from the premises using explicit rules of inference, while informal proofs mix mathematical language and natural language but should still be logically sound. [file:1][file:6]

When to use them?

Role in Computer Science

  • To verify program correctness and prove properties such as loop invariants. [file:1]
  • To reason about algorithms and data structures (e.g., termination, complexity). [file:1]
  • To design and validate logical systems like theorem provers and rule-based AI. [file:6]
Methods

Overview of Proof Methods

Method Pattern Main idea
Direct proof show \(p \to q\) assume p is true and derive q by algebra and known facts. [file:1][file:6]
Indirect proof show \(\lnot q \to \lnot p\) prove the contrapositive, which is logically equivalent. [file:1][file:6]
Proof by contradiction assume \(p \land \lnot q\) derive a contradiction with premises or known facts, forcing \(p \to q\) to be true. [file:1]
Proof by cases \(p_1 \lor \dots \lor p_n \to q\) prove q separately under each possible case \(p_i\). [file:1]
Equivalence proof show \(p \leftrightarrow q\) prove both directions \(p \to q\) and \(q \to p\). [file:1]
Step-by-step

Direct Proof of an Implication

Problem type

Prove a statement “if p, then q” about integers, reals, or other structures. [file:1][file:6]

Step-by-step:

  1. Rewrite the statement in symbolic form \(p \to q\); identify hypothesis p and conclusion q. [file:1][file:6]
  2. Assume p is true (for arbitrary but fixed objects, e.g. integer n). [file:1][file:6]
  3. Use definitions (e.g., even, odd), algebra, and known lemmas to manipulate expressions. [file:1][file:6]
  4. Gradually transform the hypothesis into the desired form of q. [file:1][file:6]
  5. Conclude that since p implies q for an arbitrary object, the implication holds in general. [file:1][file:6]
Example: To prove “If n is odd, then \(n^2\) is odd”, write n as \(2k+1\), compute \(n^2\), and show it has the form \(2m+1\) for some integer m. [file:1][file:6]
Step-by-step

Indirect Proof (Contrapositive)

Problem type

Prove \(p \to q\) by showing \(\lnot q \to \lnot p\) instead. [file:1][file:6]

Step-by-step:

  1. Identify p and q in the original implication. [file:1][file:6]
  2. Formulate the contrapositive statement \(\lnot q \to \lnot p\). [file:1][file:6]
  3. Assume \(\lnot q\) and manipulate algebraically to show \(\lnot p\) must hold. [file:1][file:6]
  4. Conclude that since the contrapositive is true, the original implication is also true. [file:1][file:6]
Example: For the theorem “If \(3n+2\) is odd then n is odd”, prove instead that “If n is even then \(3n+2\) is even” and note this is equivalent to the original statement. [file:1][file:6]
Contradiction & Cases

Proof by Contradiction and Proof by Cases

Proof by contradiction:

  1. Assume both the hypothesis p and the negation of the conclusion \(\lnot q\). [file:1]
  2. Use algebraic manipulation and known facts to derive a statement that contradicts premises or an established theorem. [file:1]
  3. Conclude that the assumption \(p \land \lnot q\) is impossible, so \(p \to q\) must hold. [file:1]
Example: Again for “If \(3n+2\) is odd then n is odd”, assume \(3n+2\) is odd and n is even, derive that \(3n+2\) is even, which contradicts the assumption. [file:1]

Proof by cases:

  1. Express the hypothesis as a disjunction \(p_1 \lor \dots \lor p_n\). [file:1]
  2. Treat each case \(p_i\) separately, assuming it true and proving q. [file:1]
  3. Once all cases imply q, conclude \(p_1 \lor \dots \lor p_n \to q\). [file:1]
Example: To prove \(|x||y| = |xy|\), split into four cases depending on signs of x and y (both non‑negative, one negative, both negative) and verify equality holds in each case. [file:1]
Failing to cover all possible cases, or using overlapping/incomplete partitions, leads to incorrect proofs by cases. [file:1]
Rules of inference

Formal Proofs with Inference Rules

Rules of inference, such as Modus Ponens, Modus Tollens, Addition, Simplification, and Hypothetical Syllogism, justify each step of a formal derivation. [file:6]

Example rule (Modus Ponens): from p and \(p \to q\), infer q; the pattern corresponds to the tautology \((p \land (p \to q)) \to q\). [file:6]
Example application: from premises “if we take a canoe trip then we will be home by sunset” and “we take a canoe trip”, conclude “we will be home by sunset”. [file:6]

Decision tree

Choosing a Proof Method

Q1: Is the statement already in the form “for all x, if condition then conclusion”?
Yes → consider direct or contrapositive
Q2: Is the conclusion a negative claim (“… is not …”, “no …”)?
Yes → contrapositive or contradiction often fits well
Q3: Does the hypothesis naturally split into disjoint cases (e.g., sign of a number)?
Yes → use proof by cases
Q4: Are you trying to show two statements are equivalent?
Yes → prove each direction separately, or transform using equivalences
